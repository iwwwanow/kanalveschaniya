# Testing: Telegram Music Bot

## Сценарии

### 1. Скачивание одного трека

**Шаги:**
1. Отправить боту ссылку на одиночный трек (напр. SoundCloud)
2. Дождаться ответа бота

**Ожидаемый результат:**
- Бот отвечает "Добавлено в очередь"
- Через некоторое время бот присылает аудио-файл (mp3)
- Трек появляется в приватном канале

**Статус:** [ ] не проверен

---

### 2. Скачивание плейлиста

**Шаги:**
1. Отправить боту ссылку на плейлист (напр. SoundCloud set)
2. Дождаться ответа бота

**Ожидаемый результат:**
- Бот отвечает "Добавлено в очередь"
- Бот сообщает сколько треков в плейлисте и что они добавлены в очередь
- Каждый трек поочерёдно присылается пользователю
- Все треки появляются в приватном канале

**Статус:** [ ] не проверен

---

### 3. Повторное скачивание одного трека

**Шаги:**
1. Отправить боту ту же ссылку на трек, который уже был скачан ранее

**Ожидаемый результат:**
- Бот отвечает "Уже есть, пересылаю..."
- Бот мгновенно пересылает трек из канала (без повторного скачивания)
- yt-dlp не запускается

**Статус:** [ ] не проверен

---

### 4. Повторное скачивание плейлиста

**Шаги:**
1. Отправить боту ту же ссылку на плейлист, который уже был скачан ранее

**Ожидаемый результат:**
- Бот разворачивает плейлист
- Каждый трек, который уже есть в кэше, пересылается из канала без скачивания
- Новые треки (если плейлист пополнился) встают в очередь на скачивание

**Статус:** [ ] не проверен

---

## Будущие сценарии

- Трек > 50MB — пропуск с уведомлением
- YouTube видео — скачивание как mp4
- Недоступная ссылка — уведомление об ошибке, retry до 3 раз
- `/status` — показывает очередь пользователя
- Два пользователя запрашивают один трек одновременно
